### 2.2 ex3.py — SVM 基础实验

#### 2.2.1 高斯核函数 `gaussian_kernel`

```python
def gaussian_kernel(x1, x2, gamma):
    return np.exp(-gamma * np.sum((x1 - x2) ** 2))
```

**思路**：高斯（RBF）核函数用于衡量两个样本之间的相似度，公式为：

$$K(x_1, x_2) = \exp\left(-\gamma \cdot \|x_1 - x_2\|^2\right)$$

其中 $\gamma$ 控制核的宽度。$\gamma$ 越大，核函数的"作用范围"越窄，两个样本需要越相近才能有较高相似度。对应的，若以 $\sigma$ 表示核宽度，正确关系为 $\gamma = \dfrac{1}{2\sigma^2}$。

#### 2.2.2 决策边界可视化 `visualize_boundary`

**思路**：根据核函数类型采用不同的绘图策略：

- **线性核**：SVM 的决策边界是超平面 $\mathbf{w}^\top\mathbf{x} + b = 0$，对于二维特征可以解析出直线方程：
  $$y = -\frac{w_0}{w_1} x - \frac{b}{w_1}$$
  直接用 `coef_` 和 `intercept_` 计算并绘制。

- **RBF 核**：决策边界是非线性曲线，无法用解析式表达。做法是在特征空间内构造一个密集的 meshgrid 网格，对每个网格点调用 `predict()` 得到类别标签，然后用等高线 `contour` 画出 0/1 的分界线。

#### 2.2.3 Part1 — 线性可分 SVM

**思路**：线性 SVM 的核心超参数是惩罚系数 $C$，它控制**间隔**与**误分类**之间的权衡：

- **$C$ 小**（如 $C=1$）：允许更多误分类，决策边界尽量使间隔最大化，对异常点不敏感，泛化能力更好。
- **$C$ 大**（如 $C=100$）：几乎不允许误分类，决策边界被迫拟合所有训练点，间隔变窄，对噪声点敏感。

实验中分别训练 $C=1$ 和 $C=100$ 的线性 SVM，对比两者决策边界的差异，直观地理解 $C$ 的作用。

#### 2.2.4 Part2 — 非线性 SVM（高斯核）

**思路**：当数据不是线性可分时，通过核技巧（Kernel Trick）将原始特征隐式映射到更高维空间，在高维空间中找到线性分界面。回到原始特征空间，该分界面即为非线性的决策边界。

实验使用 $C=1$、$\gamma=100$ 的 RBF 核 SVM，$\gamma$ 极大意味着核宽度极小，每个支持向量只影响极近的区域，决策边界能够紧密地包围每个样本簇。

#### 2.2.5 Part3 — 超参数搜索

**思路**：SVM 性能依赖于 $C$ 和 $\gamma$ 两个关键超参数，手动调参效率低，使用**网格搜索**（Grid Search）在候选值的笛卡尔积中系统地寻找最优组合。

候选值集合均为 `[0.01, 0.03, 0.1, 0.3, 1, 3, 10, 30]`，共 $8 \times 8 = 64$ 种组合，通过 `sklearn.model_selection.GridSearchCV` 自动完成交叉验证评估。

实验进行了两轮搜索：
1. 仅在 dataset3 训练集（211样本）上搜索，找到最优参数后在同一数据上绘制决策边界。
2. 将 dataset3 训练集与验证集（411样本）合并后搜索，利用更多数据找到更可靠的参数。

`dataset3_params_ver3` 函数返回最优参数字典以及对应的已训练 SVM 模型（`best_params_` 与 `best_estimator_`），可直接用于绘图。

### 2.3 ex3_spam.py — 垃圾邮件分类实验

#### 2.3.1 邮件预处理 `email_preprocess`

**思路**：原始邮件文本包含大量与内容无关的噪声（HTML标签、URL、数字等），直接使用会严重干扰模型。预处理流程如下：

```
原始邮件
  → 1. 统一小写
  → 2. 去除 HTML 标签（<...>）
  → 3. 数字 → 统一替换为 "number"
  → 4. URL → 统一替换为 "httpaddr"
  → 5. 邮件地址 → 统一替换为 "emailaddr"
  → 6. "$" 符号 → 统一替换为 "dollar"
  → 7. 按特殊字符集分词（正则 split）
  → 8. Porter 词干提取（running→run, emails→email）
  → 9. 映射到1899词词表，得到词索引列表
```

第 3–6 步的**词归一化**是关键：不同的数字、URL、邮件地址在语义层面等价，统一替换既压缩了特征空间维度，又保留了"邮件中包含数字/链接"这一垃圾邮件信号。

第 8 步的**词干提取**将词形变体归并（如 "clicking"、"clicked" 均归为 "click"），减少词表规模，提升特征复用率。

#### 2.3.2 特征提取 `feature_extraction`

**思路**：将变长的词索引列表转化为固定维度的特征向量（1899维）。采用**二值词袋模型（Binary Bag-of-Words）**：

$$x_i = \begin{cases} 1 & \text{如果词 } i \text{ 出现在邮件中} \\ 0 & \text{否则} \end{cases}$$

这种表示忽略词频（只记录"有没有"而非"出现几次"），对垃圾邮件检测效果良好——垃圾邮件通常不会通过重复某个词来逃避检测。

#### 2.3.3 Part3 — SVM 训练与评估

**思路**：使用 $C=0.1$ 的线性 SVM。$C$ 较小意味着允许一定程度的误分类，有助于防止过拟合到训练集中的噪声邮件。

训练集规模：4000 封邮件（1813 封垃圾 + 2187 封正常）；测试集：1000 封邮件。

训练后，按 `coef_` 权重排序，分别输出**权重最低的15个词**（普通邮件特征）和**权重最高的15个词**（垃圾邮件特征），帮助理解模型的决策依据。

#### 2.3.4 SpamVerify — 验证权重可解释性

**思路**：SVM 的权重向量 `coef_` 直接反映了各特征对分类决策的贡献，但这只是模型内部的"数学意义"。`SpamVerify` 函数从**数据分布**角度验证权重的可解释性：

对于权重最高/最低的词，统计其在 spam 和 ham 邮件中的实际出现频率，计算 spam/ham 出现率之比：

```
词       spam出现率   ham出现率   spam/ham比
click      0.423       0.031      13.6x
remov      0.335       0.027      12.4x
...
```

若权重高的词在 spam 中出现率确实远大于 ham（比率远大于1），则验证了 SVM 权重具有合理的语义解释。

#### 2.3.5 Part4 — 实际邮件分类预测

**思路**：加载预训练的线性 SVM 模型（`linear_svm.svm`，通过 pickle 序列化保存），对 6 封样本邮件逐一进行分类：

```
EmailSample/hamTest.txt:     普通邮件
EmailSample/spamTest.txt:    垃圾邮件
EmailSample/emailSample1.txt: 普通邮件
...
```

每封邮件经过与训练时相同的 `email_preprocess → feature_extraction` 流程，提取 1899 维特征向量后输入模型预测。

## 三、提高训练

### 3.1 Bug 1：gamma 与 sigma 的对应关系错误

#### 错误位置

**`OriginCode/ex3.py`** 中，三处参数搜索函数均错误地将 sigma 换算为 gamma：

```python
# dataset3_params_ver3，第51行
gammas = map(lambda x: 1.0 / x, sigma_values)  # 错误：gamma = 1/sigma

# dataset2_params_ver2，第78行
rbf_svm.set_params(gamma=1.0 / sigma)           # 错误：gamma = 1/sigma

# params_search，第108行
rbf_svm.set_params(gamma=1.0 / sigma)           # 错误：gamma = 1/sigma
```

此外，`part2()` 第182行的注释也是错误的：

```python
rbf_svm = svm.SVC(C=1, kernel='rbf', gamma=1.0 / sigma)
# gamma is actually inverse of sigma   ← 注释本身也是错的
```

#### 错误原因

RBF 核函数的数学定义有两种等价表示：

$$K(x_1, x_2) = \exp\!\left(-\frac{\|x_1 - x_2\|^2}{2\sigma^2}\right) \quad \text{（以核宽度 } \sigma \text{ 表示）}$$

$$K(x_1, x_2) = \exp\!\left(-\gamma \cdot \|x_1 - x_2\|^2\right) \quad \text{（sklearn 的定义）}$$

对比两式可得正确换算公式：

$$\boxed{\gamma = \frac{1}{2\sigma^2}}$$

而原始代码使用的是 $\gamma = \dfrac{1}{\sigma}$，缺少了分母中的系数 2 和 $\sigma$ 的平方，导致 $\gamma$ 被大幅高估。例如当 $\sigma = 2$ 时：

| 公式 | 计算结果 |
|------|---------|
| 错误：$\gamma = 1/\sigma = 1/2$ | $\gamma = 0.5$ |
| 正确：$\gamma = 1/(2\sigma^2) = 1/8$ | $\gamma = 0.125$ |

补充代码 `ex3.py` 的 `part2()` 中已给出正确值并加注释（第162行）：
```python
gamma = 0.125  # 对应原 sigma=2: gamma=1/(2*2^2)=0.125
```

#### 验证方式

运行 `SVM/test/sklearnGamm.py` 进行验证：

```python
from sklearn.metrics.pairwise import rbf_kernel
import numpy as np

x1 = np.array([[1, 2, 1]])
x2 = np.array([[0, 4, -1]])

gamma = 0.5
result = rbf_kernel(x1, x2, gamma=gamma)  # sklearn 计算结果 ≈ 0.011109

# 手动验证 exp(-gamma * ||x1-x2||^2)
sq_dist = 9  # ||x1-x2||^2 = 1+4+4 = 9
manual = np.exp(-gamma * sq_dist)          # exp(-0.5*9) ≈ 0.011109
```

两者结果完全相等，确认 sklearn 使用的是 $\exp(-\gamma \cdot \|x-y\|^2)$ 形式，从而推导出 $\gamma = 1/(2\sigma^2)$。

#### 修改方案

最直接的修改：将参数搜索中的 `sigma_values` 直接替换为 `gamma_values`，不再做换算，直接用 gamma 作为搜索参数：

```python
# 修改前（错误）
sigma_values = [0.01, 0.03, 0.1, 0.3, 1, 3, 10, 30]
gammas = map(lambda x: 1.0 / x, sigma_values)
parameters = {'kernel': ('rbf',), 'C': c_values, 'gamma': list(gammas)}

# 修改后（正确）
gamma_values = [0.01, 0.03, 0.1, 0.3, 1, 3, 10, 30]
parameters = {'kernel': ('rbf',), 'C': c_values, 'gamma': gamma_values}
```

若必须保留 sigma 参数（如沿用原有接口），应改为：
```python
rbf_svm.set_params(gamma=1.0 / (2 * sigma ** 2))  # 正确换算
```

### 3.2 Bug 2：错误理解"垃圾邮件特征词"

#### 错误位置

**`OriginCode/ex3_spam.py`** `part_3()` 第 96–99 行：

```python
sorted_indices = np.argsort(linear_svm.coef_, axis=None)
for i in sorted_indices[0:15]:         # 取权重【最小】的15个
    print(reversed_vocab_list[i])
```

从上下文来看，这段代码的意图是输出"垃圾邮件特征词"，但实际上 `sorted_indices[0:15]` 取的是 SVM 权重**最小（最负）**的 15 个词，这些词实际上是**普通邮件（ham）的特征词**，而非垃圾邮件词。

#### 错误原因

线性 SVM 对二分类（spam=1，ham=0）训练后，`coef_[i]` 代表特征 $i$ 对分类决策的贡献：

- `coef_[i] > 0`（正权重）：特征 $i$ 出现时，模型倾向于预测为**正类（spam）**
- `coef_[i] < 0`（负权重）：特征 $i$ 出现时，模型倾向于预测为**负类（ham）**

`np.argsort` 返回从小到大排序的下标：
- `sorted_indices[0:15]`：权重最**小**（最负）的 15 个特征 → **ham 特征词**
- `sorted_indices[-15:]`：权重最**大**（最正）的 15 个特征 → **spam 特征词**

#### 验证方式

运行补充代码中的 `SpamVerify()` 函数，对权重最高的 5 个词和权重最低的 5 个词，统计其在 spam/ham 中的实际出现频率，输出示例如下：

```
词            spam出现率   ham出现率   spam/ham比
click           0.423       0.031      13.6x  (w=正)
remov           0.335       0.027      12.4x  (w=正)
pleas           0.402       0.053       7.6x  (w=正)
...
wrote           0.021       0.198       0.1x  (w=负)
below           0.019       0.187       0.1x  (w=负)
```

从结果可以清晰看到：
- 权重为**正**的词（spam特征词）在 spam 中的出现频率远高于 ham（比率 >> 1）
- 权重为**负**的词（ham特征词）在 ham 中的出现频率远高于 spam（比率 << 1）

这进一步证明了原始代码 `sorted_indices[0:15]` 输出的是 ham 词，而非 spam 词。

#### 修改方案

将 `sorted_indices[0:15]` 改为 `sorted_indices[-15:]`，同时建议同时输出两类词以便对比：

```python
# 修改前（错误，输出的是ham特征词）
for i in sorted_indices[0:15]:
    print(reversed_vocab_list[i])

# 修改后（正确）
print("=== 普通邮件特征词（权重最低）===")
for i in sorted_indices[0:15]:
    print('{0}: {1:.4f}'.format(reversed_vocab_list[i], linear_svm.coef_[0][i]))

print("=== 垃圾邮件特征词（权重最高）===")
for i in sorted_indices[-15:]:
    print('{0}: {1:.4f}'.format(reversed_vocab_list[i], linear_svm.coef_[0][i]))
```

## 四、实验心得

### 4.1 遇到的问题与解决方案

#### 问题一：文件路径错误导致程序无法运行

`OriginCode` 中的代码使用了相对路径，例如 `'vocab.txt'`、`'emailSample1.txt'`、`'dataset_1.mat'` 等，但实际文件分别位于 `EmailSample/` 和 `datasetSVM/` 子目录下。

**解决方案**：统一在路径前添加目录前缀，修改为 `'EmailSample/vocab.txt'`、`'datasetSVM/dataset_1.mat'` 等，并在整个实验中保持从 `SVM/` 目录下运行脚本。

#### 问题二：正则表达式分词产生大量空字符串

原始代码中邮件分词使用的正则为：
```python
re.split('[ ' + re.escape("@$/#.-:&*+=[]?!(){},'\">_<;%\n") + ']', email_contents)
```
末尾是 `']'`，不带量词，遇到连续特殊字符（如 `--` 或 `  `）时，每两个相邻字符之间都会产生一个空字符串 token，导致后续处理出现大量空词。

**解决方案**：将结尾改为 `']+'`，使一个或多个连续的特殊字符被视为单个分隔符：
```python
re.split('[ ' + re.escape("@$/#.-:&*+=[]?!(){},'\">_<;%\n") + ']+', email_contents)
```
修改后额外用列表推导式过滤残余空字符串：`tokens = [t for t in tokens if t]`。

#### 问题三：gamma 与 sigma 换算错误排查过程

在调试参数搜索时，发现不同 sigma 值下 SVM 的决策边界变化趋势与预期不符——例如 sigma=10 时模型应该非常平滑，但实际边界却很复杂。怀疑 gamma 的换算有问题，查阅 sklearn 文档后发现：

> `gamma` defines how much influence a single training example has. The larger gamma is, the closer other examples must be to be affected.

这说明 gamma 越大，核宽度越小。对照 RBF 核的数学公式，手动推导出 $\gamma = \dfrac{1}{2\sigma^2}$。结合 `test/sklearnGamm.py` 中的实验验证，最终确认原始代码的 $\gamma = 1/\sigma$ 是错误的。

#### 问题四：SVM 训练时间过长

训练集有 4000 条样本、1899 维特征，每次运行 `ex3_spam.py` 都需要重新训练 SVM，消耗约 60–90 秒，严重影响调试效率。

**解决方案**：引入 `isTrainMode` 标志和 pickle 模型序列化，首次训练后将模型保存到 `linear_svm.svm`，后续运行时直接加载：
```python
if isTrainMode:
    linear_svm = svm.SVC(C=0.1, kernel='linear')
    linear_svm.fit(X, y.ravel())
    pickle.dump(linear_svm, open("linear_svm.svm", "wb"))
else:
    linear_svm = pickle.load(open("linear_svm.svm", "rb"))
```
这一改动将调试时的启动时间从 90 秒降低到不足 1 秒。

### 4.2 实验感悟与理解

**关于 SVM 参数的直觉**

通过对比 $C=1$ 和 $C=100$ 的实验结果，我对正则化参数 $C$ 有了更直观的认识：$C$ 不只是一个数字，它代表了对"间隔"和"误分类"的不同哲学取舍。$C$ 小时模型"宽容"，允许一些点越过边界换取更大间隔；$C$ 大时模型"严格"，为了零误分类而将边界压窄，容易产生过拟合。类似地，$\gamma$ 越大，每个样本的影响范围越小，模型越容易过拟合。

**关于核技巧的理解**

非线性 SVM 实验让我对核技巧有了更深的理解：我们并不需要真正地将数据映射到高维空间，只需要能够计算高维空间中的内积（即核函数值）。这一"隐式映射"的思想是 SVM 能够高效处理复杂分类问题的关键。

**关于词干提取的作用**

在处理垃圾邮件特征时，我注意到词表中存储的不是原始词，而是词干（如 `click`、`pleas`、`remov`），这使得"clicking"、"clicked"、"clicks"都映射到同一特征，大幅减少了特征稀疏性。从 SpamVerify 的结果来看，这些词干形式确实能有效区分 spam 和 ham，说明词干提取对于词袋模型的特征质量有显著提升。

**关于调试方法的体会**

这次实验中多次遇到"代码可以运行但结果不对"的情况（如 gamma/sigma 错误、spam词输出成了ham词），纯粹依靠看代码很难发现。通过编写专项验证函数（`SpamVerify`、`sklearnGamm.py`）对中间结果进行检验，能够更快定位问题。这提醒我在以后的开发中，对关键的数学公式和数据方向（升序/降序）都应当添加单元验证，而不是想当然地认为是对的。
